### Task 3.1: Fix label query bug and phase number extraction

**Files:**
- `packages/cli/src/hooks/maxsim-statusline.ts`

**Action:**
Read the full file. The background refresh script is embedded as a template literal string inside `spawnBackgroundRefresh`. The current script has these bugs:

1. **Label query bug (line ~80):** The `gh api` call uses `labels=phase:` (with a trailing colon). The label is named `phase` (no colon). Fix: change `labels=phase:` to `labels=phase`.

2. **Phase number extraction:** After fixing the label query, the current code tries to extract the phase number from a label name (`phaseRaw.replace('phase:', '')`). The research finding is that phase number should be parsed from the issue title instead, matching the pattern `[Phase XX]` in the issue title. Fix: change the gh api query to also fetch the issue title (already in the JSON response as `.title`), then parse the phase number using a regex `/^\[Phase\s+(\S+)\]/` applied to the title field. The label query still uses `labels=phase` to filter to phase issues — but the number comes from the title.

Updated query structure:
```js
const phaseRaw = execSync(
  'gh api "repos/' + owner + '/' + repo + '/issues?state=open&labels=phase&per_page=1&sort=updated&direction=desc" --jq ".[0].title"',
  { encoding: 'utf8', timeout: 10000, stdio: ['pipe', 'pipe', 'pipe'] }
).trim();
// Parse [Phase XX] from title
const titleMatch = phaseRaw.match(/^\[Phase\s+(\S+)\]/);
if (titleMatch) {
  phaseNumber = titleMatch[1];
}
// Also capture issue number for the board column query (Task 3.2)
const issueRaw = execSync(
  'gh api "repos/' + owner + '/' + repo + '/issues?state=open&labels=phase&per_page=1&sort=updated&direction=desc" --jq ".[0].number"',
  { encoding: 'utf8', timeout: 10000, stdio: ['pipe', 'pipe', 'pipe'] }
).trim();
const issueNumber = issueRaw ? parseInt(issueRaw, 10) : null;
```

Note: these two API calls can be combined into a single `gh api` call that returns both `.number` and `.title` from the JSON — use `--jq ".[0] | {number: .number, title: .title}"` and parse the JSON result.

### Task 3.2: Add board column GraphQL query and extend ProgressCache

**Files:**
- `packages/cli/src/hooks/maxsim-statusline.ts`

**Action:**
1. Extend the `ProgressCache` interface to add `board_column: string | null`:

  phase_number: string | null;

  milestone_pct: number;

2. In the background refresh script, after the phase issue number is obtained (from Task 3.1), add a GraphQL call to fetch the board column. **Use `execSync` with `input` option** to pipe the query via stdin, avoiding all shell escaping issues:

```js
let boardColumn = null;
if (issueNumber) {
  try {
    const gqlQuery = '{ repository(owner: "' + owner + '", name: "' + repo + '") { issue(number: ' + issueNumber + ') { projectItems(first: 5, includeArchived: false) { nodes { fieldValueByName(name: "Status") { ... on ProjectV2ItemFieldSingleSelectValue { name } } } } } } }';
    const boardRaw = execSync(
      'gh api graphql -f query=@-',
      { input: gqlQuery, encoding: 'utf8', timeout: 10000, stdio: ['pipe', 'pipe', 'pipe'] }
    ).trim();
    const boardData = JSON.parse(boardRaw);
    const nodes = boardData?.data?.repository?.issue?.projectItems?.nodes || [];
    if (nodes.length > 0 && nodes[0]?.fieldValueByName?.name) {
      boardColumn = nodes[0].fieldValueByName.name;
    }
  } catch (e) {
    // Graceful degrade if read:project scope missing or no project board
    boardColumn = null;
  }
}
```

**Important:** The `input` option of `execSync` pipes the string to stdin. The `@-` flag tells `gh` to read the query from stdin. This avoids all heredoc and single-quote escaping issues in the `node -e` context.

3. Update the cache write at the end of the background script to include `board_column: boardColumn`.
