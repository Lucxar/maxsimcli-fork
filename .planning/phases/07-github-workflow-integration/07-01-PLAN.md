<objective>
Enhance existing MCP tools and create new MCP tools required for GitHub-first workflow integration. This is the TypeScript foundation that all workflow markdown files will consume.
</objective>

<execution_context>
Phase 7 wires GitHub MCP tools into every MAXSIM workflow. This plan builds the tool layer. The GitHub adapter code (client.ts, issues.ts, projects.ts, mapping.ts) already exists from Phase 2 -- this plan enhances it with missing capabilities identified by research.

Key locked decisions:
- WIRE-01: GitHub-first write order (write to GitHub, abort if fails, no local fallback)
- WIRE-07: Rollback via close with state_reason: 'not_planned' (GitHub REST has no delete API)
- WIRE-06: External edit detection via body hash (not updated_at, which has false positives)
</execution_context>

**2. issues.ts -- Enhance existing functions:**

a) `closeIssue()` -- Add `stateReason` parameter (default `'completed'`):
```
export async function closeIssue(
  issueNumber: number,
  reason?: string,
  stateReason: StateReason = 'completed',

```
Use the `stateReason` parameter in the `octokit.rest.issues.update()` call instead of hardcoded `'completed'`.

b) `getPhaseIssue()` -- Expand return type to include `updated_at`, `labels`, `comments_url`:
```
Promise<GhResult<{
  number: number; id: number; title: string; state: string; body: string;
  updated_at: string; labels: string[]; comments_url: string;
}>>
```
Map `response.data.labels` to string array (handle both string and object label formats).
Map `response.data.updated_at` and `response.data.comments_url` from the response.

c) `listPhaseSubIssues()` -- Add `updated_at` to return type:
```
Array<{ number: number; id: number; title: string; state: string; updated_at: string }>
```

**3. mapping.ts -- Add body hash helpers:**

a) Add `import { createHash } from 'node:crypto';` at top
b) Add helper function:
```typescript
export function hashBody(body: string): string {
  return createHash('sha256').update(body).digest('hex');
}
```
c) Add `updatePhaseBodyHash(cwd: string, phaseNum: string, bodyHash: string)` function that loads mapping, sets `mapping.phases[phaseNum].body_hash = bodyHash`, and saves.

**2. github-tools.ts -- Add NEW tools (register inside `registerGitHubTools`):**

a) `mcp_post_comment` -- Generic comment posting:
```typescript
server.tool('mcp_post_comment', 'Post a comment on a GitHub issue.', {
  issue_number: z.number().describe('GitHub issue number'),
  body: z.string().describe('Comment body (markdown)'),
  type: z.enum(['research', 'context', 'summary', 'verification', 'uat', 'general']).optional()
    .describe('Comment type for structured header')
}, async ({ issue_number, body, type }) => { ... })
```
Call the existing `postComment()` from issues.ts. If `type` is provided, prefix body with `<!-- maxsim:type=${type} -->` HTML comment for later retrieval.

b) `mcp_batch_create_tasks` -- Create multiple task sub-issues with rollback:
```typescript
server.tool('mcp_batch_create_tasks', 'Create multiple task sub-issues for a phase with automatic rollback on failure.', {
  phase_number: z.string(),
  parent_issue_number: z.number(),
  tasks: z.array(z.object({
    task_id: z.string(),
    title: z.string(),
    body: z.string(),
  })),
}, async ({ phase_number, parent_issue_number, tasks }) => { ... })
```
Implementation pattern:
- Track `created: { issueNumber: number; taskId: string }[]`
- For each task, call `createTaskSubIssue()` + `updateTaskMapping()`
- On failure: iterate `created` in reverse, call `closeIssue(num, '[MAXSIM-ROLLBACK] Partial batch failure', 'not_planned')` for each
- Return `{ succeeded: [...], failed: [...], rolled_back: [...] }`

c) `mcp_detect_external_edits` -- Body hash comparison:
```typescript
server.tool('mcp_detect_external_edits', 'Check if a phase issue body was modified outside MAXSIM.', {
  phase_number: z.string(),
}, async ({ phase_number }) => { ... })
```
Implementation: load mapping, get stored `body_hash`. Call `getPhaseIssue()` live, compute `hashBody()` on live body. Compare. Return `{ modified: boolean, phase_number, issue_number }`.
