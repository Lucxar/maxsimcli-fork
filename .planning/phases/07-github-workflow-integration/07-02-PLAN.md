<objective>
Update init context assembly functions to include GitHub mapping data (issue numbers, project board number, github_ready flag) so downstream workflows can call MCP tools with the correct parameters.
</objective>

<execution_context>
All 21 init functions currently read exclusively from local `.planning/` files. Phase 7 requires workflows to have GitHub issue numbers and project board data available at init time. The mapping cache (github-issues.json) stores this data for fast local lookup. This plan adds GitHub data to the init context without changing the init functions' local-file-reading nature -- it reads from the mapping cache, not from live GitHub API.

Locked decisions:
- WIRE-03: Init requires GitHub setup (mandatory, no local-only mode)
- WIRE-05: Live reads are done by workflows/MCP tools, not by init context
</execution_context>

**2. init.ts -- Update init functions to populate GitHub fields:**

a) Add import at top:
```typescript
import { loadMapping } from '../github/mapping.js';
```

b) Add helper function:
```typescript
function getGitHubContext(cwd: string): {
  github_ready: boolean;
  project_number: number | null;
  mapping: IssueMappingFile | null;
} {
  try {
    const mapping = loadMapping(cwd);
    if (!mapping || !mapping.project_number) {
      return { github_ready: false, project_number: null, mapping: null };
    }
    return { github_ready: true, project_number: mapping.project_number, mapping };
  } catch {
    return { github_ready: false, project_number: null, mapping: null };
  }
}
```
