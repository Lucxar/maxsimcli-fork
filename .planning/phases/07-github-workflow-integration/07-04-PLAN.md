```markdown
### Stage Detection (GitHub-First)

Detect planning stage by querying GitHub Issue state:

1. Get the phase issue number from init context (`phase_issue_number`)
2. If no phase_issue_number exists, phase has not been set up on GitHub yet
   - Call `mcp_create_phase_issue` to create it (uses roadmap data)
3. Call `mcp_get_issue` to read the phase issue body and comments
4. Check issue comments for existing artifacts:
   - Has `<!-- maxsim:type=context -->` comment? → Context/discuss stage complete
   - Has `<!-- maxsim:type=research -->` comment? → Research stage complete
   - Has `## Plan` comment (from mcp_post_plan_comment)? → Planning stage complete
5. Determine next stage:
   - No context comment → run discuss stage (plan-discuss.md)
   - Has context, no research → run research stage (plan-research.md)
   - Has context + research (or research skipped), no plan → run create stage (plan-create.md)
   - Has plan comment → planning complete, report status
```

Also update any references to reading/writing local CONTEXT.md, RESEARCH.md files -- these now come from/go to GitHub.
