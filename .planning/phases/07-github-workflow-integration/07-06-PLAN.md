Read both files fully first.

**1. go.md -- Switch decision tree to live GitHub queries:**

Replace local state detection with GitHub-based detection:

a) **Next action detection:** Instead of checking local files:
  - Call `mcp_get_all_progress` to get current state of all phases
  - Call `mcp_detect_interrupted` to check for interrupted phases
  - Determine next action based on:
    - Any interrupted phases? → Resume execution
    - Any phases in "In Progress" with incomplete tasks? → Continue execution
    - Any phases in "To Do"? → Plan next phase
    - All phases "Done"? → Milestone complete
  - Dispatch to the appropriate command

b) **Keep local reads for context:** Still read STATE.md for blockers, decisions. Still read ROADMAP.md for phase ordering and dependencies.

**2. verify-phase.md -- Post verification results to GitHub:**

Replace local VERIFICATION.md and UAT.md writing with GitHub posting:

a) **Verification results:** After running verification checks:
  - Build verification content in memory (same structured format)
  - Call `mcp_post_comment` with `type: 'verification'` on the phase issue
  - Include: truths checked, artifacts verified, key links validated, pass/fail status

b) **UAT results:** After user acceptance testing:
  - Call `mcp_post_comment` with `type: 'uat'` on the phase issue
  - Include: scenarios tested, results, user feedback

c) **Board transition on verification pass:**
  - If verification passes and PR is merged: call `mcp_move_issue` to move phase to "Done"
  - Call `mcp_close_issue` on the phase issue

d) **Board transition on verification fail:**
  - Keep phase in "In Review" or move back to "In Progress" depending on severity
  - Post failure details as comment for gap closure

e) **Remove local file writes:** Remove instructions to write VERIFICATION.md or UAT.md to `.planning/phases/` directories.
