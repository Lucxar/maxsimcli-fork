<action>
**Strategy:** Use `npx tsc --noEmit` as the guide. After Task 1 renamed all functions, TypeScript will flag every call site that needs updating. Work through the errors file by file.

**General pattern for each consumer file:**

1. If the file has a function that calls a now-async core function (like `loadConfig`, `safeReadFile`, etc.), that calling function MUST become async
2. Add `await` before every call to the renamed functions
3. Change the function's return type from `T` to `Promise<T>`
4. Propagate async up the call chain: if function A calls function B which now returns Promise, function A must await B and become async too

**Critical file: init.ts (1060 lines, 15+ functions)**

This is the largest consumer. Every `cmdInit*` function uses sync core utilities. Convert each one:
- Change `export function cmdInitXxx(...)` to `export async function cmdInitXxx(...)`
- Change return type from `CmdResult` to `Promise<CmdResult>`
- Add `await` before every `loadConfig()`, `findPhaseInternal()`, `safeReadFile()`, `pathExistsInternal()`, `listSubDirs()`, `getArchivedPhaseDirs()`, `getRoadmapPhaseInternal()`, `getMilestoneInfo()` call
- This is mechanical: find each function call, add `await` before it

**Critical file: cli.ts (handlers)**

The `cli.ts` handlers call `cmdInit*` functions. Since those now return `Promise<CmdResult>` instead of `CmdResult`:
- Every handler that calls a `cmdInit*` function must `await` the result before passing to `handleResult()`
- The handler functions in cli.ts are already within an async context (async main()), so adding await is safe
- BEWARE Pitfall 5: If a handler does `const result = cmdInitFoo(...)` without await, `result` is a Promise (truthy) -- it looks like success but has no data

**MCP tool files (phase-tools.ts, context-tools.ts, config-tools.ts):**

These files define MCP tool handlers that are already async. They just need `await` added before core function calls:
- `const config = loadConfig(cwd)` -> `const config = await loadConfig(cwd)`
- `const phase = findPhaseInternal(cwd, p)` -> `const phase = await findPhaseInternal(cwd, p)`

**Other consumer files (frontmatter, drift, state, context-loader, artefakte, skills, verify, template, commands):**

Same mechanical pattern. For each file:
1. Find all calls to the 10 renamed functions
2. Add `await` before each call
3. Make the enclosing function async if it isn't already
4. Update the return type to Promise<...>
5. Propagate async up to any callers within the same file
