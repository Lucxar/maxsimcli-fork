**Step 1: Rewrite github-tools.ts.**

The current file has 29K+ of MCP tool registrations that use `detectGitHubMode()` and return "local-only mode" when gh isn't available. Rewrite to:

a) Remove ALL `detectGitHubMode()` / `'local-only'` checks. Replace with a `requireAuth()` call at the start of each tool handler. If `requireAuth()` throws `AuthError`, catch it and return a structured error with setup instructions.

b) Replace all `ghExec()` / `ghGraphQL()` calls with the new adapter functions:
   - `createPhaseIssue()`, `createTaskSubIssue()`, `postPlanComment()` from issues.ts
   - `closeIssue()`, `reopenIssue()`, `listPhaseSubIssues()` from issues.ts
   - `ensureLabels()` from labels.ts
   - `ensureMilestone()` from milestones.ts

c) Update imports: remove all `../github/gh-legacy.js` imports. Import from `../github/index.js`.

d) Each tool handler should follow this pattern:
```typescript
try {
  requireAuth();
} catch (e) {
  if (e instanceof AuthError) {
    return { content: [{ type: 'text', text: `GitHub auth required: ${e.message}` }] };
  }
  throw e;
}
```

e) Review each tool function. Remove tools that are no longer relevant (e.g., tools that managed local-only issue tracking). Keep tools that provide value with the new GitHub-native approach.
