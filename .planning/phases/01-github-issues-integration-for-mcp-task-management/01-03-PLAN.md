<action>
Create the issue CRUD module. This is the largest single module in the GitHub integration layer. All functions return GhResult.

**Issue Creation Functions:**

1. **createTaskIssue(opts: { title: string; phaseNum: string; planNum: string; taskId: string; summary: string; actions: string[]; acceptanceCriteria: string[]; dependencies?: number[]; milestone?: string; projectTitle?: string; labels?: string[]; estimate?: number }): Promise&lt;GhResult&lt;{ number: number; url: string; node_id: string }&gt;&gt;**

   Build issue body with this format:
   ```
   ## Summary
   {summary}

   <details>
   <summary>Full Specification</summary>

   ### Actions
   - {action1}
   - {action2}

   ### Acceptance Criteria
   - [ ] {criterion1}
   - [ ] {criterion2}

   ### Dependencies
   Depends on: #{dep1}, #{dep2}

   </details>

   ---
   *Phase: {phaseNum} | Plan: {planNum} | Task: {taskId}*
   *Generated by MAXSIM*
   ```

   Title format: `[P{phaseNum}] {title}`

   Run `gh issue create --title "..." --body "..." --label maxsim --label phase-task [--milestone "..."] [--project "..."]`.

   CRITICAL: `gh issue create` outputs a URL to stdout, NOT JSON. Parse the URL to extract the issue number: `parseInt(url.split('/').pop())`. Then fetch full details via `gh issue view {number} --json nodeId,number,url` to get node_id.

   Return `{ number, url, node_id }`.

2. **createParentTrackingIssue(opts: { phaseNum: string; phaseName: string; childIssueNumbers: number[]; milestone?: string; projectTitle?: string }): Promise&lt;GhResult&lt;{ number: number; url: string; node_id: string }&gt;&gt;**

   Title: `[Phase {phaseNum}] {phaseName}`
   Body:
   ```
   ## Phase {phaseNum}: {phaseName}

   ### Tasks
   - [ ] #{child1}
   - [ ] #{child2}
   - [ ] #{child3}

   ---
   *Phase tracking issue — Generated by MAXSIM*
   ```

   Labels: `maxsim`, `phase-task`. Same URL parsing as createTaskIssue.

3. **createTodoIssue(opts: { title: string; description?: string; acceptanceCriteria?: string[]; milestone?: string; projectTitle?: string }): Promise&lt;GhResult&lt;{ number: number; url: string; node_id: string }&gt;&gt;**

   Lighter body: just description and optional acceptance criteria (no `<details>` collapse).
   Labels: `maxsim`, `todo`.

**Lifecycle Functions:**

4. **closeIssue(issueNumber: number, reason?: 'completed' | 'not_planned'): Promise&lt;GhResult&lt;void&gt;&gt;** — `gh issue close {number} [--reason ...]`.

5. **reopenIssue(issueNumber: number): Promise&lt;GhResult&lt;void&gt;&gt;** — `gh issue reopen {number}`.

6. **closeIssueAsSuperseded(oldIssueNumber: number, newIssueNumber: number): Promise&lt;GhResult&lt;void&gt;&gt;** — Post comment "Superseded by #{newIssueNumber}", add `superseded` label, close as completed. On the new issue, post comment "Replaces #{oldIssueNumber}".

7. **postComment(issueNumber: number, body: string): Promise&lt;GhResult&lt;void&gt;&gt;** — `gh issue comment {number} --body "..."`.

**Import Function:**

8. **importExternalIssue(issueNumber: number): Promise&lt;GhResult&lt;{ number: number; title: string; labels: string[] }&gt;&gt;** — Read existing issue via `gh issue view {number} --json title,labels,body,state`. Add `maxsim` and `imported` labels. Return issue details for AI to decide placement.

**Utility Functions:**

9. **getIssueBranchName(issueNumber: number, slug: string): string** — Return `maxsim/issue-{issueNumber}-{slug}`. Pure function, no async needed.

10. **buildPrBody(closesIssues: number[], additionalContent?: string): string** — Build PR description body with `Closes #{N}` lines for each issue. This enables auto-close on merge (AC-08).

11. **updateParentTaskList(parentIssueNumber: number, childIssueNumber: number, checked: boolean): Promise&lt;GhResult&lt;void&gt;&gt;** — Read parent issue body, find the `- [ ] #{childNumber}` or `- [x] #{childNumber}` line, toggle checkbox. Update via `gh issue edit {number} --body "..."`. This keeps the parent tracking issue task list in sync.

Parse URLs from `gh issue create` stdout by trimming whitespace and extracting the last path segment. Handle edge case where stdout might have trailing newline.

<action>
Add batch issue creation to the existing issues.ts module (extend, do not rewrite).

**Add these exported functions:**

1. **createAllPlanIssues(opts: { phaseNum: string; planNum: string; phaseName: string; tasks: Array&lt;{ taskId: string; title: string; summary: string; actions: string[]; acceptanceCriteria: string[]; dependencies?: string[]; estimate?: number }&gt;; milestone?: string; projectTitle?: string; cwd: string }): Promise&lt;GhResult&lt;{ parentIssue: number; taskIssues: Array&lt;{ taskId: string; issueNumber: number }&gt; }&gt;&gt;**

   Orchestrates eager issue creation on plan finalization (locked decision: all issues created at once):
   a. Create all task issues with concurrency limit of 5 (to avoid GitHub's 80/min secondary rate limit — see RESEARCH.md Pitfall 5). Use a simple sequential loop with batches of 5, or Promise.all with a pool.
   b. After all task issues created, create parent tracking issue with all child issue numbers.
   c. Update mapping file for all created issues.
   d. Return parent issue number and all task issue numbers.

   For dependency handling: resolve task dependency IDs to issue numbers from the just-created issues. E.g., if task 1.2 depends on task 1.1, look up 1.1's issue number from the batch results.

2. **supersedePlanIssues(opts: { phaseNum: string; oldPlanNum: string; newPlanNum: string; newIssueNumbers: Array&lt;{ taskId: string; issueNumber: number }&gt;; cwd: string }): Promise&lt;GhResult&lt;void&gt;&gt;**

   When a plan is re-planned (fresh issues per plan):
   a. Load mapping to find old plan's issue numbers.
   b. For each old issue, call closeIssueAsSuperseded with the corresponding new issue.
   c. Close old parent tracking issue as superseded.
   d. Update mapping: mark old entries with `superseded_by` field, add new entries.

Handle partial failures gracefully — if some issues in a batch fail, continue with the rest and report which failed.
</action>
