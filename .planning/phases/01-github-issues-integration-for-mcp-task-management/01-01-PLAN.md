<task id="1.2" type="auto">
<files>
packages/cli/src/github/gh.ts
</files>
<action>
Create the core gh CLI wrapper module. Implements 4 exported functions:

1. **checkGhAuth(): Promise&lt;AuthStatus&gt;** — Run `gh auth status` via execFile. Parse stderr output (gh writes to stderr, not stdout). Detect: installed (ENOENT = not installed), authenticated (exit code 1 = not auth'd), scopes (parse "Token scopes:" line), username (parse "Logged in to ... as USERNAME"). Return AuthStatus object. Timeout: 10 seconds.

2. **detectGitHubMode(): Promise&lt;GitHubMode&gt;** — Call checkGhAuth(). Return `'full'` only if installed AND authenticated AND hasProjectScope. Otherwise return `'local-only'`. Log warning to stderr if installed but missing project scope: "GitHub Projects requires 'project' scope. Run: gh auth refresh -s project".

3. **ghExec&lt;T&gt;(args: string[], options?: { cwd?: string; parseJson?: boolean; timeout?: number }): Promise&lt;GhResult&lt;T&gt;&gt;** — Execute `gh` with given args via execFile. Default timeout 30s. If parseJson is true (or if args contain `--json` or `--format`), parse stdout as JSON. Map exit codes to GhErrorCode: ENOENT -> NOT_INSTALLED, exit 1 -> check stderr for auth/permission patterns, exit 4 -> NOT_FOUND. For `gh issue create` specifically: do NOT try to parse JSON (it returns a URL string). Return `{ ok: true, data }` or `{ ok: false, error, code }`.

4. **ghGraphQL&lt;T&gt;(query: string, variables?: Record&lt;string, unknown&gt;): Promise&lt;GhResult&lt;T&gt;&gt;** — Build args: `['api', 'graphql', '-f', 'query=...']`. For each variable, add `-F` `key=value` (use `-F` not `-f` for non-string types). Parse JSON response. Check for `errors` array in GraphQL response and convert to GhResult error.

Import types from `./types.js`. Use `node:child_process` execFile with promisify. Do NOT import octokit or any npm package.

Error message handling: always include the raw stderr in the error string so the AI can react intelligently to specific GitHub API errors (rate limits, 404s, permission issues).
</action>
