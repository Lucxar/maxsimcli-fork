The workflows currently reference old agent names like `maxsim-executor`, `maxsim-planner`, `maxsim-verifier`, `maxsim-phase-researcher`, `maxsim-code-reviewer`, `maxsim-spec-reviewer`, etc. All of these must be updated to the new 4 agent names.

The installer currently handles agents/ and skills/ but not rules/. Must add rules/ directory installation.

IMPORTANT: When updating workflow agent references, also update the spawn prompt patterns. Old pattern: "First, read ~/.claude/agents/maxsim-planner.md for your role and instructions." This must change because:
1. Agent files are now at ~/.claude/agents/planner.md (no maxsim- prefix)
2. BUT -- Claude Code's native agent system uses the Agent tool with agent name, NOT manual file reading. The workflows should spawn using `subagent_type="planner"` and Claude Code loads the agent automatically.
3. However, review each workflow to understand if it uses the native Agent tool pattern or manual file reading, and update accordingly.

Also important: Old specializations (maxsim-code-reviewer, maxsim-spec-reviewer, maxsim-debugger, etc.) now map to the generic `verifier` agent. Orchestrator spawn prompts must carry richer context to differentiate. For example, where a workflow currently spawns `maxsim-code-reviewer`, it should now spawn `verifier` with a spawn prompt that says "Review this implementation for code quality" and suggests the `code-review` skill.
