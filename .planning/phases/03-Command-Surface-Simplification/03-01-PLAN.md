**Step 1: Create the command template** at `templates/commands/maxsim/plan.md`.

Follow the exact format pattern from `execute-phase.md` and `plan-phase.md`:
- Frontmatter with `name: maxsim:plan`, `description`, `argument-hint: "[phase-number] [--force-research] [--skip-verify]"`, `allowed-tools` (Read, Write, Bash, Glob, Grep, Task, WebFetch, mcp__context7__*)
- `<objective>` block: "Execute the plan state machine: Discussion -> Research -> Planning. Each stage produces artifacts, shows a gate summary, and waits for user confirmation before advancing."
- `<execution_context>` block: `@./workflows/plan.md` and `@./references/ui-brand.md`
- `<context>` block: "$ARGUMENTS" with flag descriptions
- `<process>` block: "Execute the plan workflow from @./workflows/plan.md end-to-end."

Flags:
- `--force-research` -- Re-run research even if RESEARCH.md exists
- `--skip-verify` -- Skip plan verification loop after planning stage
- No `--gaps` flag (gap closure is handled differently in the new system)

**Step 2: Create the thin orchestrator workflow** at `templates/workflows/plan.md`.

This file MUST be under 20KB (target ~12-15KB). It is the state machine orchestrator that:

1. **Initialize** -- Call `node .claude/maxsim/bin/maxsim-tools.cjs init plan-phase "$PHASE"` to get phase state JSON. Extract `has_context`, `has_research`, `plan_count`, `phase_dir`, `phase_number`, `phase_name`.

2. **Detect current stage** based on artifacts:
   - `plan_count > 0` -> Already planned (re-entry flow)
   - `has_research == true && plan_count == 0` -> Start at Planning stage
   - `has_context == true && has_research == false` -> Start at Research stage
   - None of the above -> Start at Discussion stage

3. **Re-entry flow** (when already planned):
   Show status summary (plan count, wave structure, last planned date).
   Offer options: (1) View existing plans, (2) Re-plan from scratch, (3) Execute phase via `/maxsim:execute`, (4) Done.
   Wait for user choice. If re-plan: delete existing plans and restart from Discussion.

4. **Discussion stage** (delegate to sub-workflow):
   Reference `@./workflows/plan-discuss.md` for full discussion logic.
   After discussion completes, show gate:
   ```
   ## Gate: Discussion Complete
   **Captured:** {N} decisions across {M} areas
   **Locked decisions:** {bullet list from CONTEXT.md}
   **Claude's discretion:** {list}
   **Context file:** {path to CONTEXT.md}
   Continue to research? [Yes / Review context / Re-discuss area]
   ```
   Wait for user confirmation. If "Review context": display CONTEXT.md contents. If "Re-discuss": loop back.

5. **Research stage** (delegate to sub-workflow):
   Reference `@./workflows/plan-research.md` for research logic.
   After research completes, show gate:
   ```
   ## Gate: Research Complete
   **Key findings:** {3-5 bullets from RESEARCH.md}
   **Confidence:** {HIGH/MEDIUM/LOW}
   **File:** {path to RESEARCH.md}
   Continue to planning? [Yes / Review research / Re-research]
   ```
   Wait for user confirmation.

6. **Planning stage** (delegate to sub-workflow):
   Reference `@./workflows/plan-create.md` for planning + verification logic.
   After planning completes, show gate:
   ```
   ## Gate: Planning Complete
   **Plans:** {N} plan(s) in {M} wave(s)
   **Wave structure:** {wave summary table}
   **Files:** {list of PLAN.md paths}
   Ready to execute? Run /maxsim:execute {phase_number}
   ```

7. **Checkpoint before /clear**: If context is getting full (>70%), recommend `/clear` and post checkpoint comment to GitHub Issue using `mcp_post_plan_comment`. Checkpoint format:
   ```markdown
   ## MAXSIM Checkpoint
   **Command:** /maxsim:plan
   **Stage:** {current_stage} ({stage_num}/3)
   **Completed:** {list of completed stages with summaries}
   **Resume from:** {next stage}
   ```

The workflow must NOT contain the full discussion, research, or planning logic inline. It delegates via @references to the three sub-workflows. The orchestrator is ONLY: init, detect stage, delegate, show gate, advance.

Important: Do NOT use the `AskUserQuestion` tool in gate confirmations. Use natural conversation (plain text output + wait for response). The old discuss-phase workflow used AskUserQuestion but the new system uses conversational gates.

5. Write CONTEXT.md with locked decisions, Claude's discretion, and deferred ideas
6. Return control to orchestrator (do NOT show gate -- orchestrator handles gates)

Key changes from the old discuss-phase.md workflow:
- Remove the "offer next steps" section at the end -- orchestrator handles routing
- Remove any references to `/maxsim:plan-phase` or `/maxsim:research-phase` -- this IS part of /maxsim:plan now
- Keep the domain-aware gray area generation (the user-SEE/CALL/RUN/READ/ORGANIZE pattern)
- Keep the probing depth pattern (4 questions per area, check, 4 more if needed)
- Use natural conversation instead of `AskUserQuestion` tool for all user interaction
- Keep the CONTEXT.md template and output format from the existing workflow

**File 2: `templates/workflows/plan-research.md`** (target ~8-12KB)

Adapt logic from `templates/workflows/research-phase.md` (2KB) and the research step in `templates/workflows/plan-phase.md` (18KB). This file contains the full research stage:

1. Resolve researcher model: `node .claude/maxsim/bin/maxsim-tools.cjs resolve-model maxsim-phase-researcher --raw`
2. Check if RESEARCH.md already exists AND `--force-research` flag is NOT set -> skip research, return to orchestrator
3. Spawn `maxsim-phase-researcher` agent via Task tool with phase context
4. Handle researcher return -- validate RESEARCH.md was created
5. Return control to orchestrator

The researcher agent prompt already contains all the research logic. This sub-workflow just needs to:
- Assemble the context for the researcher agent (phase info, ROADMAP.md, REQUIREMENTS.md, CONTEXT.md)
- Spawn the agent
- Validate the output

**File 3: `templates/workflows/plan-create.md`** (target ~15-18KB)

Adapt logic from `templates/workflows/plan-phase.md` (18KB) -- specifically the planning and verification steps. This file contains the full planning stage:

1. Resolve planner and checker models
2. Spawn `maxsim-planner` agent via Task tool with full phase context (ROADMAP.md, REQUIREMENTS.md, CONTEXT.md, RESEARCH.md, STATE.md)
3. Handle planner return -- validate PLAN.md files were created
4. If `--skip-verify` flag is NOT set: spawn `maxsim-plan-checker` agent to verify plans
5. If checker finds issues: re-spawn planner with revision context (max 2 iterations)
6. Commit plan files if `commit_docs` is true
7. Return control to orchestrator

Key changes from the old plan-phase.md workflow:
- Remove the research step (it is now a separate stage handled by plan-research.md)
- Remove the "offer next steps" routing at the end -- orchestrator handles routing
- Keep the planner spawn pattern, checker verification loop, and commit logic
- Keep the structured return parsing from planner agent

For ALL three files:

- Do NOT reference old command names (/maxsim:plan-phase, /maxsim:discuss-phase, /maxsim:research-phase)
- Each file should be self-contained: it receives phase context and returns when its stage is done
