<action>
Rewrite the `review_cycle` step in execute-plan.md to add retry counters and escalation. The current step has 4 stages (Spec Review, Code Review, Simplify, Final Review). Keep this structure but add:

**1. Retry counter tracking at step start:**
```
SPEC_ATTEMPTS=0
CODE_ATTEMPTS=0
SIMPLIFY_ATTEMPTS=0
FINAL_ATTEMPTS=0
MAX_REVIEW_ATTEMPTS=3
```

**2. For Stage 1 (Spec Review) — wrap in retry loop:**
- Before spawning verifier: increment `SPEC_ATTEMPTS`
- After result: if FAIL and SPEC_ATTEMPTS < MAX_REVIEW_ATTEMPTS, fix issues and re-run (existing behavior but now counted)
- After result: if FAIL and SPEC_ATTEMPTS >= MAX_REVIEW_ATTEMPTS, escalate:
  ```
  ## Review Escalation: Spec Review Failed After 3 Attempts

  **Stage:** Spec Review
  **Attempts:** 3/3
  **Last failure reason:** {failure details from verifier}

  The spec review has failed 3 times. This may indicate a fundamental mismatch between the plan spec and the implementation.

  **Options:**
  1. Fix manually and type "retry" to re-run spec review
  2. Type "override" to skip spec review (will be flagged in SUMMARY.md)
  3. Type "abort" to stop execution
  ```
  Wait for user response. If "retry": reset SPEC_ATTEMPTS to 0, loop. If "override": mark as OVERRIDDEN in review tracking, proceed. If "abort": stop.

**3. For Stage 2 (Code Review) — same retry + escalation pattern.**

**4. For Stage 3 (Simplify) — add config gate:**
Before running simplify, check config:
```bash
SIMPLIFY_ENABLED=$(node ~/.claude/maxsim/bin/maxsim-tools.cjs config-get review.simplify_review 2>/dev/null || echo "true")
```
If `SIMPLIFY_ENABLED` is "false": skip Stage 3 and Stage 4 entirely. Log: "Simplify stage skipped (disabled in config)".

**5. For Stage 4 (Final Review) — same retry + escalation pattern.**

**6. Review cycle timing:**
Record start time before Stage 1, end time after last stage. Per-stage: record stage start/end. Format: `{stage}: {result} ({attempts} attempt(s), {duration}s)`.

**7. Update the review tracking section at the end:**
Replace the existing review cycle markdown table with:
```markdown
## Review Cycle

| Stage | Result | Attempts | Duration |
|-------|--------|----------|----------|
| Spec Review | {PASS|FAIL|OVERRIDDEN} | {N}/3 | {X}s |
| Code Review | {APPROVED|BLOCKED|OVERRIDDEN} | {N}/3 | {X}s |
| Simplify | {CLEAN|FIXED|BLOCKED|SKIPPED} | {N}/3 | {X}s |
| Final Review | {APPROVED|BLOCKED|SKIPPED|N/A} | {N}/3 | {X}s |

**Total review time:** {total}s
**Escalations:** {count} ({details or "None"})
```

Keep all existing Task() spawn prompts for verifier and executor agents unchanged — only add the retry/escalation wrapper logic around them.
