**1. Create `maxsim-sync-reminder.ts`** -- a new PostToolUse hook for detecting `.planning/` file changes.

The hook receives PostToolUse JSON on stdin with fields: `session_id`, `cwd`, `hook_event_name`, `tool_name`, `tool_input` (with `file_path`), `tool_response`.

Implementation:

```typescript
interface SyncReminderInput {
  session_id?: string;
  cwd?: string;
  tool_input?: { file_path?: string };
}

interface SyncReminderOutput {
  hookSpecificOutput: {
    hookEventName: string;
    additionalContext: string;
  };
}
```

Logic for `processSyncReminder(data: SyncReminderInput): SyncReminderOutput | null`:
1. Extract `session_id` and `tool_input.file_path` from input. If either is missing, return null.
2. Normalize `file_path` using `path.normalize()` (Windows backslash handling -- see research pitfall #5).
3. Check if the normalized path contains `${path.sep}.planning${path.sep}` or ends with `${path.sep}.planning`. If not, return null silently.
4. Load debounce state from temp file: `path.join(os.tmpdir(), 'maxsim-sync-${session_id}.json')`.
   State shape: `{ callsSinceRemind: number; reminded: boolean }`.
5. If state file does not exist, create initial state `{ callsSinceRemind: 0, reminded: false }`.
6. Increment `callsSinceRemind`.
7. If `reminded` is false (first .planning/ write) OR `callsSinceRemind >= 10` (debounce expired): reset counter to 0, set `reminded: true`, write state, return the reminder output.
8. Otherwise: write updated state, return null.

The reminder message (locked decision): `".planning/ files changed locally. Consider syncing to GitHub Issues when ready."`

Output format:
```json
{
  "hookSpecificOutput": {
    "hookEventName": "PostToolUse",
    "additionalContext": ".planning/ files changed locally. Consider syncing to GitHub Issues when ready."
  }
}
```

Include a standalone entry at the bottom:
```typescript
if (require.main === module) {
  readStdinJson<SyncReminderInput>((data) => {
    const result = processSyncReminder(data);
    if (result) {
      process.stdout.write(JSON.stringify(result));
    }
  });
}
```

Import `readStdinJson` from `./shared`. Import `fs`, `os`, `path` from `node:` prefixed modules.

**2. Update `maxsim-check-update.ts`** -- add backup logic.

Add a new exported function `createBackupBeforeUpdate(cwd: string): string | null` that:
1. Determines the source directory: `path.join(cwd, CLAUDE_DIR)` (the current install).
2. Creates backup directory at `path.join(cwd, CLAUDE_DIR, 'maxsim-backup')`.
3. Copies key MAXSIM directories to the backup: `commands/maxsim/`, `maxsim/`, `hooks/`, `agents/`, `skills/`. Use recursive copy for each (check if source dir exists before copying).
4. Writes a `backup-meta.json` inside the backup dir with `{ created: ISO timestamp, version: installed version from VERSION file }`.
5. Returns the backup directory path on success, null on failure.
6. Wrap everything in try/catch -- backup failure should not block the update.

Do NOT call this function from the SessionStart hook itself. The backup is triggered by the install process (when user runs `npx maxsimcli@latest`), not by the hook. The hook only checks for updates and caches the result. Export the function so the installer can use it.
